# Contributing

Kogniq welcomes future contributions from humans and AI agents. It is a planned AI Learning Intelligence Platform whose first reference domain will be GATE. The project is currently in repository-planning stage; contributions must respect the active roadmap stage and preserve the boundary between platform capabilities and examination-specific plugins.

Maintainer: [Yash Mishra](https://github.com/yash-mishraa).

## Before Contributing

- Read `README.md` and follow `.ai/handoff.md`.
- Review architecture, decisions, technology status, coding rules, and progress.
- Confirm the proposed change has a clear owner and does not implement a future stage prematurely.
- Use the repository's GitHub issue and pull-request workflows when they become available.

## Contribution Standards

Keep changes focused, evidence-backed, documented, and testable. Preserve privacy, accessibility, licensing, and security. Do not add dependencies, generated data, model artifacts, secrets, or vendor coupling without review. Use placeholders rather than guessing manual project information.
Do not embed examination-specific curricula, terminology, policies, or branching in the platform core. New domain work requires an approved extension contract and explicit scope.

## Humans

Describe the user or engineering outcome, constraints, validation, and architectural impact. Review AI-generated work as carefully as human-written work. Do not merge changes solely because automated checks pass.

## AI Agents

Follow the mandatory context-read order, inspect existing work, state assumptions, remain within prompt scope, and update progress before handoff. Never claim tests or capabilities that were not actually verified. Do not overwrite history in ADRs or Recent Sessions.

## Proposed Workflow

1. Open an issue or agreed task describing scope and acceptance criteria.
2. Make a focused branch and commits.
3. Add or update tests and documentation as required.
4. Complete the code-review and architecture checklists.
5. Request review from affected owners.

Branch protection, issue templates, review policy, public contact channels, and release policy remain to be defined in a later stage.
