# Applications

`apps/` contains future Kogniq composition roots. Applications assemble package capabilities and expose product experiences; they do not become reusable dependency sources.

- `api/` is reserved for authoritative application workflows and external contracts.
- `web/` is reserved for the accessible browser experience.

No framework, runtime, API, or UI is implemented. See [the system blueprint](../.ai/system_blueprint.md) and [package contracts](../.ai/package_contracts.md).
